# go_max_tcp

пример сгенерированного клиента и логики взаимодействия в `main.go`

## USAGE

через go mod установить все зависимости

запуск:

### прохождение авторизации

>[!warning]
> не учитывает случай, если после числового кода не просит пароль подтверждения, нужно поправить

```
go run . auth
```

После авторизации:

```
go run . start
```

далее интерактивно проходите авторизацию
- отсутствуют проверки ввода, вводите сразу правильно
  - телефон надо вводить с `+7` кода
  - пароль нужно с первого раза ввести корректно. Также, реализован ввод через `fmt.Scanln` - пробельные символы не обрабатываются
  - пароль аналогично, не заложил attempts логику


## Баги go реализации

- некорректно сгенерировались attachments типы - они не работают. Базовый API работает, если сигнатуры совпали с фактическим поведением.
- ошибка в response opcode 128 - ttl поле имеет тип boolean, а не int
